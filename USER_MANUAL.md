# คู่มือผู้ใช้ (ฉบับละเอียดและเข้าใจง่าย)

เอกสารนี้อธิบายวิธีเตรียมเครื่อง รันเซิร์ฟเวอร์ ดูเอกสาร API และสั่งให้ระบบสแครปข้อมูลจาก Facebook/TikTok โดยไม่ต้องเขียนโค้ดมากนัก

1) สิ่งที่ต้องเตรียม
- เครื่องที่ติดตั้ง Python 3.8+ และอินเทอร์เน็ต
- คำสั่งเทอร์มินัล (Terminal / CMD)

2) ติดตั้งโปรแกรมที่จำเป็น

```bash
pip install -r requirements.txt
playwright install chromium
```

3) ตั้งค่าตัวแปรแวดล้อม (ตัวอย่าง)
- สร้างไฟล์ `.env` ในโฟลเดอร์โปรเจค (ถ้าต้องการ) และใส่ค่าที่จำเป็น เช่น:

```
API_PORT=8000
API_TOKEN=your-secret-token-here
GROQ_API_KEY=xxx
```

4) รันเซิร์ฟเวอร์

โหมดง่ายที่สุด:

```bash
python main.py
```

โหมดพัฒนา (อัพโค้ดแล้วรีโหลดอัตโนมัติ):

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

5) ตรวจสอบสถานะ

```bash
curl http://localhost:8000/health
```

6) เปิดหน้าเอกสาร API (ไม่ต้องเขียนโค้ด)

เปิดเว็บเบราว์เซอร์และไปที่: http://localhost:8000/docs

ในหน้านี้คุณจะเห็นรายการ endpoints แยกตามหมวด (`/fb`, `/tiktok` ฯลฯ) และแบบฟอร์มให้กรอกข้อมูลเพื่อทดสอบแต่ละคำขอ (Try it out)

7) ตัวอย่างการใช้งานแบบทีละขั้นตอน (ผู้ใช้ทั่วไป)

- ตัวอย่าง A — สั่งสแครปวิดีโอจาก TikTok (โดยไม่ต้องรู้ path ลึกๆ)
	1. รันเซิร์ฟเวอร์ และเปิด `http://localhost:8000/docs`
	2. หา tag `tiktok` แล้วขยาย endpoint ที่เกี่ยวข้องกับการสแครป (ชื่อที่เห็นใน UI จะบอกชัด เช่น `Scrape by keyword`)
	3. คลิก `Try it out` กรอกค่า เช่น `keyword`, `max_results` แล้วกด `Execute`
	4. ตรวจผลลัพธ์จาก response ที่แสดง หรือดูไฟล์ output ใน `scraped_data/`

- ตัวอย่าง B — สั่งสแครปโพสต์จากกลุ่ม Facebook
	1. เปิด `http://localhost:8000/docs`
	2. หา tag `fb` แล้วเลือก endpoint สำหรับดึงโพสต์จากกลุ่ม
	3. กรอกข้อมูลที่ระบบขอ (เช่น group id, limit) และกด `Execute`
	4. ผลลัพธ์จะแสดงในหน้า response และ/หรือบันทึกใน `FDATA/outputs`

8) ดูผลลัพธ์ไฟล์
- ผลลัพธ์ที่บันทึกมักเป็น JSON และเก็บใน `scraped_data/` หรือในโฟลเดอร์ผู้ใช้ที่ `FDATA/outputs`
- หาก endpoint คืน JSON เป็นผลลัพธ์ คุณสามารถคัดลอกหรือดาวน์โหลดจากหน้า Swagger หรือบันทึกจาก terminal

9) สั่งงานผ่าน `curl` (ตัวอย่างทั่วไป)

หากคุณต้องการส่งคำสั่งผ่าน terminal แทนใช้ UI ให้ค้นหา path ที่ถูกต้องจาก `/docs` แล้วใช้คำสั่ง `curl` แบบนี้ (ตัวอย่างสมมติ):

```bash
curl -X POST "http://localhost:8000/tiktok/scrape" \
	-H "Content-Type: application/json" \
	-d '{"keyword":"dance","max_results":10}'
```

หมายเหตุ: เปลี่ยน `/tiktok/scrape` เป็น path จริงที่เห็นใน `http://localhost:8000/docs`

10) ปัญหาที่พบบ่อยและการแก้ไขเบื้องต้น
- เซิร์ฟเวอร์ไม่ขึ้น: ตรวจสอบว่ารันคำสั่งในโฟลเดอร์โปรเจคและไม่มี error ขณะ start
- Playwright ไม่พบเบราว์เซอร์: รัน `playwright install chromium` อีกครั้ง
- การสแครปไม่คืนผลลัพธ์หรือได้ผลลัพธ์ไม่ครบ: ตรวจสอบ logs ใน `FDATA/logs/` และลองลด `max_results` หรือเพิ่มเวลาหน่วง
- ต้องการดูหน้าจอเบราว์เซอร์แบบมี UI: ในโค้ดให้เปลี่ยน option `headless=True` เป็น `headless=False` (นักพัฒนาจะช่วยเปลี่ยนให้)

11) คำถามที่พบบ่อย (FAQ)
- Q: ฉันต้องมีบัญชี Facebook/TikTok ไหม?
	A: ขึ้นกับ endpoint — บางฟีเจอร์อาจต้องล็อกอินด้วยโปรไฟล์ผู้ใช้ที่เก็บใน `FDATA/` หรือ `T_DATA/` ค่ะ
- Q: ข้อมูลที่ได้ปลอดภัยไหม?
	A: โปรดตรวจสอบกฎหมายและข้อตกลงการใช้งานของแพลตฟอร์มก่อนใช้งาน อย่าเก็บหรือเผยแพร่ข้อมูลส่วนบุคคลโดยไม่ได้รับอนุญาต

12) ถ้าต้องการความช่วยเหลือเพิ่มเติม
- ผมสามารถเพิ่มตัวอย่าง `curl` เฉพาะ endpoint ที่คุณต้องการ หรือสร้าง Postman collection ให้ได้ — บอกชื่อ endpoint หรือว่าต้องการ Postman/`curl`/Docker เลือกมาได้
